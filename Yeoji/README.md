# 여지 (Yeoji)

> 화엄경 십지품의 "여지(如地)" — 대지와 같이, 좋은 것과 나쁜 것을 가리지 않고 모두 품어
> 그 위에 만물이 자라게 하는 바탕. OS라는 존재의 본질과 닮았다고 생각해 이름 지었습니다.

한글 어셈블리 프로젝트 [훈닉스(Hunix)](../Yana)에서 시작된 여정이, macOS 유저스페이스를 벗어나
**OS도 커널도 없는 맨 하드웨어(bare metal) 위**로 처음 발을 뗀 프로젝트입니다.

개발 단계에서는 로마자 표기 `Yeoji`를 사용하고, 완성 단계에서는 한글 이름 `여지`를 그대로
브랜드로 사용할 예정입니다. 버전이 올라가도 이름은 바뀌지 않고, 내부 버전 번호만 갱신합니다.

## 지금 여기까지 (v0.0.1)

- `aarch64-none-elf` 타겟으로 QEMU `virt` 머신 위에 직접 부팅하는 최소 커널
- OS, 부트로더, libc 없이 `_start` → `kernel_main` 까지 순수 어셈블리 + C로 진입
- PL011 UART(`0x09000000`)에 직접 바이트를 써서 화면에 "Hello, World!" 출력 성공

```
==========================================
   Yeoji OS - v0.0.1 (aarch64, baremetal)
==========================================

  Hello, World!
  여지, 안녕!

Booted with no OS beneath us. Just Yeoji and the metal.
```

## 빌드 & 실행

```bash
clang --target=aarch64-none-elf -ffreestanding -nostdlib -c boot.S -o boot.o
clang --target=aarch64-none-elf -ffreestanding -nostdlib -c kernel.c -o kernel.o
ld.lld -T link.ld boot.o kernel.o -o kernel.elf
qemu-system-aarch64 -M virt -cpu cortex-a72 -nographic -kernel kernel.elf
```

`Ctrl+A` 다음 `X` 로 QEMU를 종료할 수 있습니다.

## 1. 링크 스크립트 (link.ld) - 설계도면

>- QEMU(가상 컴퓨터)나 진짜 컴퓨터는 전원 스위치를 똭 켜면, 약속된 방 번호(메모리주소)로 CPU 조종사를 보냄 .
>- QEMU 가상 컴퓨터는 "어이, 난 켜지면 무조건 `0x40080000`번 방부터 읽을거야! 라고 규직을 정해놓음
>- `link.ld` 는 메모리 공간에 소스코드를 어떻게 배치할 지 그리는 **도면** 임.
>- `ENTRY(_start)`: 야 CPU야 켜지자 마자 `_start` 방으로 다로 뛰어 들어가 
>- `. = 0x40080000` : "여기가 시작점이야!"
>- `KEEP(*(.text.boot))` : "첫 방(0x40080000) 에 무조건 boot.S 의 _start 코드들을 락(Lock) 걸어놔. 딴놈들이 못들어오게!"
>- `__bss_start` & `_bss_end` : "청소할 영역(BSS)의 시작과 끝 주소에 깃발을 꽂아 둔다."
>- `_stack_top` : 스택은 **하늘에서 아래로 떨어지며 쌓이는 구조**라네 
>- 그래서 16KB 공간을 확보한 뒤 그 **맨 꼭대기 주소**를 깃발로 꽂아놓은 것

## 2. 부팅 어셈블리 (boot.S) - "조종사의 첫 임무"

>- 자, 컴퓨터 전원이 켜졌네! 
>- CPU 조종사들이 `_start` 방으로 우르르 진입함 
  >- 2-1. [다중코어진압] - "대장만 남고 나머지는 자라!"
    >>- 멀티코어 컴퓨터가 일꾼 중에 0번 대장 만 남고 나머지는 재우는 단계
```assembly
_start:
	mrs x0, mpidr_el1    // 1. 자기 신분증(코어 ID가 담긴 시스템 레지스터)을 꺼내서 x0에 담음.
	and x0, x0, #0xFF    // 2. 신분증 뒷자리(코어 번호)만 쏙 뽑아냄.
	cbz x0, _master_core // 3. "어라? 0번 대장이네?" -> 대장방(_master_core)으로 가라!
_loop:
  wfe      // 4. 대장이 아니면 (1, 2, 3번 코어) -> "이벤트가 올 때까지 잠들어라(Wait For Event)!"
	b _loop  //    깨어나도 다시 재우는 무한 감옥 루프!
    
```
  >- 2-2. [.bss 메모리 광속 대청소] - "칠판 지우기"
    >>- OS는 1초에 수백 번씩 시계추를 흔들어서 멀티태스킹을 하고 있음. 
    >>- `asm_enable_timer`는 시계에 알람(인터럽트)을 맞추는 것이고, 
    >>- `enable_irq`는 `CPU`에게 "자, 이제 알람 소리(`인터럽트`) 들리게 귀 열어(`인터럽트 허용`)!" 라고 해주는 작업.
```assembly
msr cntp_tval_el0, x0  // "x0 만큼 타이머 클럭이 흐르면 삐삐빅 알람 울려라!"
msr cntp_ctl_el0, x1   //  타이머 스위치 On!
msr daifclr, #2        //  CPU의 귀를 막고 있던 마스크(DAIF)를 해제(Clear)해서 인터럽트 통신을 허용함.
```

3. 스택 주소 장착해서 C언어 메인 함수로 골인!

외계인 기술의 핵심인 **`link.ld` 문법의 미스터리**를 몇 가지만 재미있게 풀어주겠네!

### 👽 링커 스크립트 외계 구문 해석기

#### 1. 점 하나(`.`): "나 지금 어디 서 있니?" (Location Counter)
이 링커 스크립트에서 가장 골 때리는 게 바로 이 점(`.`) ㅋㅋㅋ. 
이 점은 링커가 **"메모리를 컴파일된 바이너리로 차곡차곡 채워나갈 때 현재 서 있는 위치 주소"**를 의미. 

*   `. = 0x40080000;` 
    *   **외계어 번역**: "야! 주소 측정기(Location Counter) 들고 `0x40080000` 주소 방 앞으로 순간이동해!"
*   `. = ALIGN(16);`
    *   **외계어 번역**: "지금 주소가 16으로 나누어떨어지는 예쁜 주소가 아니면, 뒤에 빈 공간을 채워서 16의 배수 주소로 이동해!" (CPU가 16바이트 단위로 데이터를 긁어갈 때 속도가 제일 빨라서 하는 짓이네)
*   `__bss_start = .;`
    *   **외계어 번역**: "지금 내가 서 있는 이 주소 번지를 `__bss_start`라는 이름표(심볼)로 선언한다! 이 이름표는 나중에 C언어나 어셈블리에서 가져다 쓸 수 있어!"

#### 2. `*` 와 `*(.text*)`: "우주의 모든 파일에서 다 긁어모아라" (Wildcards)
괄호 안의 아스테리스크(`*`)는 와일드카드 문자라네.
*   `*(.text*)`
    *   **외계어 번역**: "컴파일된 모든 파일(`*`) 중에서 이름에 `.text`가 들어가는 코드 조각들을 몽땅 긁어모아서 지금 서 있는 내 발밑(`.text` 섹션)에 차곡차곡 쌓아라!"

#### 3. `. += 0x4000;`: "우주선 주차 구역 확보!"
더하기 등호(`+=`)는 우리가 아는 그 대입 연산자가 맞네.
*   `. += 0x4000;`
    *   **외계어 번역**: "현재 내가 서 있는 주소에서 16KB(`0x4000`)만큼 주소 측정기를 뒤로 슥 밀어버려! (그 사이에 아무 코드도 배치하지 마!)" 
    *   이동해서 비어버린 16KB 공간이 바로 우리 대장 코어가 쓸 **"스택(Stack) 주차장"**이 됨..

## 왜 별도 프로젝트인가

> 기존 `Yana/`(훈닉스의 macOS 유저스페이스 실험장)와는 타겟 트리플, 바이너리 포맷,
> 실행 권한 레벨이 전부 다름.

| | Yana | Yeoji |
|---|---|---|
| 타겟 | `arm64-apple-macos` | `aarch64-none-elf` |
| 바이너리 | Mach-O | ELF |
| 실행 환경 | EL0 (유저 권한, libSystem 사용) | EL1 (커널 권한, 하드웨어 직접 제어) |

`Yana`에서 검증한 한글 어셈블리 니모닉 문법(`할당하기`→`mov`, `분기실행`→`bl` 등)이
여물면, 이 커널에 그대로 이식하는 것이 장기 목표입니다.

## QEMU 

> QEMU 가상 머신에 ARM64 어셈블리 커널 코드를 얹어 실제로 구동(부팅)할 때 
> 한 방에 각 잡을 수 있는 초경량 QEMU ARM64 부팅 명령 플래그

```bash
# M1 맥북의 하드웨어 가속(hvf)을 받아 가상 ARM64 환경을 초고속으로 띄우는 신공!
qemu-system-aarch64 \
  -M virt \
  -cpu host \
  -accel hvf \
  -m 1024 \
  -nographic \
  -kernel 무결점_커널_바이너리.bin

# 그래픽 화면까지 QEMU로 직접 띄워 테스트할 때 
# -nographic을 빼고 -display cocoas 같은 맥 전용 옵션을 달아주면 미려한 창이 똭 튀어나옴 
```
